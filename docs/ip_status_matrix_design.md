# Socks5 代理 IP 标签化管理与智能分配方案

本方案旨在解决生产环境下，Socks5 代理池如何针对不同业务场景（如 YouTube、Facebook、电商爬虫）进行 IP 精细化调度，以应对 IP 被目标平台屏蔽的问题。

---

## 一、 方案核心目标

1.  **标签化管理**：为每个 IP 维护一份针对不同目标平台（Target）的信用等级。
2.  **存活探测**：自动检测 IP 对特定目标的可用性（主动拨测 + 业务反馈）。
3.  **精准调度**：根据客户的业务属性，分配最适合的 IP，避开不可用资源。
4.  **资源最大化**：即使某个 IP 被 A 平台封禁，它依然可以被高效地分配给访问 B 平台的业务。

---

## 二、 多维度状态矩阵设计 (The Status Matrix)

这是方案的核心数据结构，记录了 IP 在不同业务维度下的状态指数。

### 1. 维度定义
*   **IP 标识 (IP ID)**：每个出口 IP。
*   **业务目标 (Target)**：如 `youtube`, `facebook`, `amazon`, `google` 等。
*   **状态详情 (Status Info)**：
    *   `Status`: 0-未知, 1-正常(OK), 2-封禁(Blocked), 3-风险(Risky)。
    *   `LastUpdate`: 最后一次检测或报告的时间。
    *   `FailCount`: 在该目标下的连续失败次数。

### 2. 存储实现 (Redis Hash)
推荐在 Redis 中为每个 IP 建立一个哈希表：
**Key**: `ip:status:{IP_ADDRESS}`

| 域 (Field) | 值 (JSON Value) | 说明 |
| :--- | :--- | :--- |
| `youtube` | `{"s": 1, "t": 1711943000, "f": 0}` | 状态正常 |
| `facebook` | `{"s": 2, "t": 1711944500, "f": 3}` | 已封禁，标记冷却中 |

## 三、 系统运作逻辑

### 1. 分配策略 (Acquire Logic)
当客户发起连接请求时，调度器执行以下逻辑：
1.  **目标匹配**：识别客户的业务类型 $T$。
2.  **池化过滤**：
    *   **优先选**：`Matrix[IP][T].Status == 1` (已知正常)。
    *   **备选**：`Matrix[IP][T].Status == 0` (从未试过，去开荒)。
    *   **跳过**：`Matrix[IP][T].Status == 2` (且处于封禁期内)。
3.  **负载均衡**：在过滤后的候选子集中，通过轮询或连接数最少策略进行分配。

### 2. 反馈回路 (The Feedback Loop)
系统必须具备自我修补能力：
*   **被动反馈**：爬虫或业务端检测到 HTTP 403 Forbidden 或 Google Captcha 关键词。通过 API(`/report_block`) 告知系统，系统立即更新矩阵中该 IP 对应 Target 的状态为 `Blocked`。
*   **主动拨测**：后台探测服务每隔一段时间通过代理出口访问 `https://www.youtube.com/favicon.ico`。若成功，将状态重置为 `OK`；若失败，累加 `FailCount`。

### 3. 冷却与恢复 (Cooldown & Recovery)
封禁状态具有时效性。
*   **策略**：进入封禁状态后，该 IP 对特定目标的分配权将被隔离 $N$ 小时（如 2 小时）。
*   **自动洗白**：隔离期过后，状态重置为 `Unknown`。下次有业务“开荒”成功后，重新标记为 `OK`。

---

## 四、 生产实施路线建议

1.  **第一步：基础架构**
    *   在分布式网关层增加 `User.Target` 属性。
    *   建立 Redis IP 状态存储集群。

2.  **第二步：调度升级**
    *   修改集群内部的 `AcquireIP` 函数，增加对 Target 参数的支持。
    *   实现带标签过滤的筛选器逻辑。

3.  **第三步：反馈闭环**
    *   开发 `/report_block` REST API。
    *   集成 Prometheus/Grafana 监控各目标平台的 IP 被封率，实现实时预警。

---

## 五、 方案优势

*   **隔离性**：YouTube 业务的封禁不会影响 Facebook 业务的 IP 质量。
*   **高效率**：告别盲目随机分配，显著提升业务抓取成功率。
*   **低成本**：通过“业务互补”，极大提高了有限 IP 资源的复用上限。
